#### Detailed Phase 3 Documentation: Dashboard Data Flow and Dependencies

**Objective:** Map dashboards to `production.db` tables, and then trace these tables back to their original CSV data sources via the `sheets_data.db` staging tables and the `unified_import.py` process.

**Unified Import Script (`scripts/unified_import.py`) Data Flow Overview:**

1.  **CSV Files (Raw Data Source):**
    *   `downloads/cmlog.csv`: Raw **coordinator task** data (e.g., `Date Only`, other task columns).
    *   `downloads/psl.csv`: Raw **provider task** data (PSL source, e.g., `DOS`, other service columns).
    *   `downloads/rvz.csv`: Raw **provider task** data (RVZ source, e.g., `DOS`, other service columns).
    *   `downloads/patients/ZMO_Main.csv` or `downloads/ZMO_MAIN.csv`: Raw **patient master data** (e.g., `LAST`, `FIRST`, `DOB`).

2.  **Staging Database (`sheets_data.db`):**
    *   `cmlog.csv` -> `sheets_data.db`.`source_coordinator_tasks_history`.
    *   `psl.csv` + `rvz.csv` -> `sheets_data.db`.`SOURCE_PROVIDER_TASKS_HISTORY`.
    *   `ZMO_Main.csv` -> `sheets_data.db`.`SOURCE_PATIENT_DATA`.

3.  **Transformation (PowerShell Scripts - `4a-transform.ps1`, `4b-transform.ps1`, `4c-transform.ps1`):**
    *   These scripts consume data from `sheets_data.db` staging tables, perform transformations, and populate `production.db` tables.

**B. Mapping `production.db` Tables to `sheets_data.db` Staging Tables to Raw CSVs:**

*   **`production.db`.`coordinator_tasks_YYYY_MM`**
    *   **Populated by:** `sheets_data.db`.`source_coordinator_tasks_history` (via transformation scripts).
    *   **Originates from:** `downloads/cmlog.csv` (`Date Only`, other task columns).

*   **`production.db`.`provider_tasks_YYYY_MM` and `production.db`.`provider_tasks`**
    *   **Populated by:** `sheets_data.db`.`SOURCE_PROVIDER_TASKS_HISTORY` (via transformation scripts).
    *   **Originates from:** `downloads/psl.csv` and `downloads/rvz.csv` (`DOS`, other service columns).

*   **`production.db`.`patients`**
    *   **Populated by:**
        1.  `sheets_data.db`.`SOURCE_PATIENT_DATA` (via transformation scripts).
        2.  `onboarding_patients` table (via `insert_patient_from_onboarding`).
    *   **Originates from:**
        1.  `downloads/patients/ZMO_Main.csv` or `downloads/ZMO_MAIN.csv` (`LAST`, `FIRST`, `DOB`).
        2.  Manual entry in `onboarding_dashboard` forms.

*   **`production.db`.`patient_panel`**
    *   **Populated by:** Derived from `patients` and `patient_assignments` (via transformations/views), and direct sync/updates from `onboarding_patients`.
    *   **Originates from:** Ultimately from `downloads/patients/ZMO_Main.csv` and manual entry in `onboarding_dashboard` forms.

*   **`production.db`.`patient_assignments`**
    *   **Populated by:** Transformation scripts (from `sheets_data.db` staging tables `staging_patient_assignments`), and manual assignments in dashboards.
    *   **Originates from:** Primarily `downloads/patients/ZMO_Main.csv` (for initial assignments/relations) and manual user actions.

*   **`production.db`.`coordinator_monthly_summary_YYYY_MM`**
    *   **Populated by:** Aggregation from `coordinator_tasks_YYYY_MM` (via transformation scripts/views).
    *   **Originates from:** Indirectly from `downloads/cmlog.csv`.

*   **`production.db`.`provider_task_billing_status`**
    *   **Populated by:** Real-time generation from `provider_tasks_YYYY_MM` and direct updates from `weekly_billing_dashboard`.
    *   **Originates from:** Indirectly from `downloads/psl.csv` and `downloads/rvz.csv`.

*   **`production.db`.`onboarding_patients`**
    *   **Populated by:** Manual entry in `onboarding_dashboard` forms and subsequent updates within the onboarding workflow.
    *   **Originates from:** Manual user input in the `onboarding_dashboard`.

*   **`production.db`.`task_billing_codes`**
    *   **Populated by:** Likely static configuration or initial import.
    *   **Originates from:** External configuration/manual seeding.

*   **`production.db`.`staff_codes`**
    *   **Populated by:** Manual configuration.

*   **`production.db`.`etl_watermarks`**
    *   **Populated by:** `unified_import.py` (internal ETL tracking).
